⚙️ Admin — impostazioni salvate sul server (clicca per aprire)
Serve Maps JavaScript API + Places API. Salvata sul server, consegnata solo dopo il pass code. Google Cloud Console →
Dopo ogni ricerca l'app controlla nel registro (deluxy-anagrafiche) se in quella città o provincia abbiamo già un partner o un prospect e lo mostra in cima ai risultati. Basta la chiave: l'URL serve solo per puntare a un registro diverso (es. http://localhost:3060 in sviluppo).
Resta solo sul server (mai nel browser): serve a /api/segnala per creare il prospect nel registro. Si genera con npm run chiave -- deluxy-search --scrittura.
Client ID OAuth (tipo Web) con l'API People attiva e origine https://search-deluxy.vercel.app. Vuoto = il pulsante scarica un file .vcf da aprire con la rubrica.
Ogni operatore entra con email + password personale e le sue azioni compaiono nello Storico con la sua email. Password vuota = invariata; riga rimossa = utenza revocata. Il pass code principale resta quello dell'amministratore (unico che può salvare le impostazioni).
Chiave creata — copiala ORA, non sarà più visibile:
Le integrazioni chiamano le API con l'header x-api-key al posto di email+password; nello Storico compaiono come chiave:<nome>. Il segreto è salvato hashato: se lo perdi, revoca e genera una chiave nuova. Salvate e revocate subito (non serve «Salva impostazioni»).
Il token Shopify resta sul server, non arriva mai al browser. Lascia il token vuoto per non modificarlo. Token: Shopify Admin → Sviluppa app → Installa → Token di accesso Admin API (inizia con shpat_).
